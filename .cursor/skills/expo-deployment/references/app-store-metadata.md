# App Store Metadata Checklist

Use this checklist before iOS release submission.

## Required metadata

- App name, subtitle, and promotional text
- Full description and keyword set
- Support URL and privacy policy URL
- Category selection and age rating questionnaire

## Assets

- App icon and launch assets aligned to current build
- iPhone/iPad screenshots for supported form factors
- Optional preview videos for key user flows

## Review readiness

- Demo account credentials (if login required)
- Notes explaining gated or region-specific features
- Accurate encryption/export compliance answers

## Quality recommendations

- Keep release notes specific and user-facing
- Match screenshots to current production UI
- Avoid placeholder copy in localized metadata
