# iOS App Store Submission Flow

## Release flow

1. Run final production build and submission via EAS
2. Wait for App Store Connect processing
3. Complete metadata, pricing, and availability
4. Submit for review
5. Release manually or automatically after approval

## Review preparation

- Verify onboarding/login works with reviewer credentials
- Ensure any subscriptions/in-app purchases are configured
- Validate privacy nutrition labels and tracking declarations
- Include clear review notes for non-obvious flows

## After approval

- Monitor crash and ANR rates post-release
- Track conversion and retention signals from the first cohort
- Roll back by submitting a new hotfix build if needed
